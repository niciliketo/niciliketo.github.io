## Introduction

[n8n](https://n8n.io) is an open-source workflow automation tool that allows you to connect various applications and services together. Unlike cloud-hosted solutions, self-hosting n8n gives you complete control over your data and workflows.

In this guide, we'll deploy n8n to a Hetzner Cloud server using Kamal, a modern deployment tool that makes it easy to deploy web applications with zero-downtime deployments and automatic SSL certificate management.

## Why This Stack?

### Hetzner Cloud
- **Cost-effective**: More affordable than AWS, DigitalOcean, or other cloud providers
- **High performance**: Excellent server specifications for the price
- **European data centers**: Great for GDPR compliance

### Kamal
- **Simple deployment**: Deploy with a single command
- **Zero-downtime**: Update your app without interrupting service
- **Automatic SSL**: Let's Encrypt certificates configured automatically
- **No complex infrastructure**: No need to learn Kubernetes or complex orchestration

### n8n
- **Open source**: Full control over your automation workflows
- **Self-hosted**: Your data stays on your server
- **Powerful**: Connect 200+ apps and services

## Prerequisites

Before you begin, you'll need:

1. **A Hetzner Cloud account** - Sign up at [hetzner.com](https://www.hetzner.com/cloud)
2. **A domain name** - You'll need a domain pointing to your server
3. **Docker Hub account** - For storing your Docker images (free account works)
4. **SSH key** - For secure server access
5. **Basic command-line knowledge** - Comfortable using the terminal

### System Requirements

- **Local machine**: macOS, Linux, or Windows with WSL2
- **Server**: Minimum 2GB RAM (4GB recommended for production)
- **Ruby 3.0+**: For running Kamal
- **Docker**: For building images locally

## Step 1: Set Up Your Hetzner Server

### Create a Server

1. Log in to [Hetzner Cloud Console](https://console.hetzner.cloud/)
2. Create a new project (e.g., "n8n-production")
3. Click "Add Server" and configure:
   - **Location**: Choose closest to your users
   - **Image**: Ubuntu 22.04 or 24.04
   - **Type**: CPX11 (2 vCPU, 2GB RAM) minimum
   - **SSH Key**: Add your public SSH key
4. Click "Create & Buy Now"

### Note Your Server Details

Once created, note:
- **Server IP address**
- **SSH access**: `root@your-server-ip`

### Test SSH Connection

```bash
ssh root@your-server-ip
```

If successful, you should get access to the server.

## Step 2: Configure Your Local Environment

### Install Ruby with mise (Recommended)

[mise](https://mise.jdx.dev/) is a modern tool version manager:

```bash
# Install mise (if not already installed)
curl https://mise.run | sh

# Install Ruby
mise install ruby@latest
mise use ruby@3.4.7

# Verify installation
ruby --version
```

### Install Kamal

```bash
gem install kamal
kamal version
```

### Create Your Project

```bash
mkdir n8n-deployment
cd n8n-deployment
git init
```

### Set Up SSH Agent

Configure SSH to use your Hetzner key:

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/your-hetzner-key
```

Test the connection:

```bash
ssh root@your-server-ip "echo 'Connection successful!'"
```

## Step 3: Create the n8n Dockerfile

Create a `Dockerfile` in your project root:

```dockerfile
FROM n8nio/n8n:latest

ENV NODE_ENV=production
ENV N8N_PORT=5678
ENV N8N_PROTOCOL=https
ENV N8N_HOST=your-domain.com

EXPOSE 5678
```

Replace `your-domain.com` with your actual domain.

## Step 4: Configure Kamal

Create `config/deploy.yml`:

```yaml
service: n8n
image: your-dockerhub-username/n8n

servers:
  web:
    - your-server-ip

registry:
  username: your-dockerhub-username
  password:
    - KAMAL_REGISTRY_PASSWORD

env:
  clear:
    PORT: 5678
  secret:
    - N8N_BASIC_AUTH_ACTIVE
    - N8N_BASIC_AUTH_USER
    - N8N_BASIC_AUTH_PASSWORD
    - N8N_ENCRYPTION_KEY
    - N8N_PROTOCOL
    - N8N_HOST

volumes:
  - /root/.n8n:/home/node/.n8n

builder:
  arch: amd64

proxy:
  ssl: true
  host: your-domain.com
  app_port: 5678
```

Replace:
- `your-dockerhub-username` with your Docker Hub username
- `your-server-ip` with your Hetzner server IP
- `your-domain.com` with your domain

## Step 5: Set Up Environment Variables

### Create .env.example Template

```bash
# .env.example
# Docker Registry Password (Docker Hub)
KAMAL_REGISTRY_PASSWORD=your-docker-hub-password-here

# n8n Authentication
N8N_BASIC_AUTH_ACTIVE=true
N8N_BASIC_AUTH_USER=admin
N8N_BASIC_AUTH_PASSWORD=generate-a-secure-password

# n8n Configuration
N8N_PROTOCOL=https
N8N_HOST=your-domain.com
N8N_PORT=5678

# n8n Encryption Key (DO NOT CHANGE after first deployment)
N8N_ENCRYPTION_KEY=generate-with-openssl-rand-hex-32
```

### Create Your .env File

```bash
# Copy the template
cp .env.example .env

# Generate encryption key
openssl rand -hex 32
```

Edit `.env` and fill in:
- Your Docker Hub password
- A secure basic auth password
- The generated encryption key
- Your domain name

### Set Up .gitignore

Create `.gitignore` to protect your secrets:

```bash
# Environment variables (contains secrets)
.env

# Kamal secrets
.kamal/secrets

# n8n data
.n8n/

# OS files
.DS_Store
```

### Create Kamal Secrets File

```bash
mkdir -p .kamal
cp .env .kamal/secrets
```

## Step 6: Deploy to Hetzner

### Prepare the Server

First, create the n8n data directory on your server:

```bash
ssh root@your-server-ip "mkdir -p /root/.n8n && chmod 777 /root/.n8n"
```

### Commit Your Configuration

```bash
git add Dockerfile config/ .gitignore .env.example
git commit -m "Add Kamal deployment configuration for n8n"
```

### Run Initial Setup

Start your SSH agent and deploy:

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/your-hetzner-key

# First-time deployment
kamal setup
```

This will:
1. Verify Docker is installed
2. Build and push your n8n Docker image
3. Set up kamal-proxy for SSL
4. Deploy the n8n container
5. Configure automatic SSL certificates

The deployment takes 2-5 minutes. Watch for "Deployed" at the end!

## Step 7: Configure DNS

### Add DNS Records

In your domain registrar's DNS settings, add an A record:

```
Type: A
Name: n8n (or your subdomain)
Value: your-server-ip
TTL: 300
```

### Wait for Propagation

DNS can take 5-30 minutes to propagate. Check status:

```bash
# Test DNS resolution
nslookup n8n.your-domain.com

# Should return your server IP
```

### SSL Certificate

Once DNS propagates, Kamal's proxy will automatically:
1. Request a Let's Encrypt SSL certificate
2. Configure HTTPS
3. Redirect HTTP to HTTPS

## Access Your n8n Instance

Visit `https://n8n.your-domain.com` in your browser.

### First-Time Setup

1. You'll see the n8n setup wizard
2. Create your owner account:
   - **Email**: Your email address
   - **Password**: A secure password (different from basic auth)
3. Complete the setup

**Note**: The basic auth credentials in `.env` protect access to the site, while the n8n account is for logging into the application.
